# Student Notes Manager

_A Full-Stack CRUD Web Application_

---

## 1. Problem Statement

Students frequently store notes across multiple platforms, leading to disorganization and data loss.  
The goal of this project is to design and implement a **full-stack web application** that allows a user to **create, read, update, and delete notes** through a React frontend and a Node.js backend using a RESTful API.

The system must clearly separate frontend and backend concerns while maintaining consistent data flow and reliability.

## 3. Functional Requirements

### 3.1 Data Model

**Entity: `Note`**

| Field     | Type      | Required | Description                |
| --------- | --------- | -------- | -------------------------- |
| id        | string    | Yes      | Unique identifier          |
| title     | string    | Yes      | Short title of the note    |
| content   | string    | Yes      | Main note body             |
| createdAt | timestamp | Yes      | Time note was created      |
| updatedAt | timestamp | Yes      | Time note was last updated |

> Thought: Why should timestamps be generated by the backend?

### 3.2 Backend Requirements (Node + Express)

#### API Endpoints (CRUD)

| Method | Endpoint     | Description              |
| ------ | ------------ | ------------------------ |
| POST   | `/notes`     | Create a new note        |
| GET    | `/notes`     | Retrieve all notes       |
| GET    | `/notes/:id` | Retrieve a specific note |
| PUT    | `/notes/:id` | Update an existing note  |
| DELETE | `/notes/:id` | Delete a note            |

#### Behavioral Rules

- Use JSON for all requests and responses
- Validate incoming data
- Return proper HTTP status codes
- Handle missing or invalid resources gracefully

### 3.3 Frontend Requirements (React)

#### Core Features

- Display all notes
